# ADR-004: gRPC для Go↔Python IPC

**Статус:** Active  
**Дата:** 2026-07-17  
**Контекст:** Ранее предполагалось, что Go↔Python коммуникация будет через gRPC (protobuf) для передачи данных OCR/CV. После удаления OCR из проекта, Python-сервис в runtime больше не нужен.  
**Решение:** gRPC между Go и Python не реализуется. Python используется только в офлайн ML-контуре (обучение моделей), без runtime-коммуникации с Go.  
**Последствия:**  
  - Positive: меньше движущихся частей; проще архитектура; нет protobuf toolchain.  
  - Negative: модель обучения и рантайм-адвайзер на разных языках — экспорт модели требует продуманного формата.  
**Связанные ADR:** ADR-005 (remove HTTP/WS), ADR-003 (superseded)
