# ADR-008: SQLite как runtime-хранилище

**Статус:** Active  
**Дата:** 2026-07-17  
**Контекст:** Требуется локальное хранилище для логов матчей, снапшотов волн, рекомендаций и кэша справочных данных API.  
**Решение:** SQLite через `modernc.org/sqlite` (pure Go, без CGo). WAL journal mode для конкурентного чтения/записи.  
**Последствия:**  
  - Positive: zero dependency (no CGo); один файл БД — легко бекапить/переносить; WAL позволяет читать во время записи.  
  - Negative: не подходит для высоконагруженной аналитики (не наша задача); нет сетевого доступа (и не нужно).  
**Связанные ADR:** ADR-009 (API as source of truth)
