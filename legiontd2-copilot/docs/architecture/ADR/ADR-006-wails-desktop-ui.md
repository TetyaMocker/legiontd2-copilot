# ADR-006: Wails как Desktop UI

**Статус:** Active  
**Дата:** 2026-07-17  
**Контекст:** Текущий UI — браузерная SPA (HTML/CSS/JS). ТЗ (UX1) требует отдельное окно, не overlay. Wails v2 уже указан в конфигурации проекта (`wails.json`), но не реализован.  
**Решение:** Использовать Wails v2 для нативного Desktop-окна. Wails позволяет писать UI на HTML/CSS/JS (или React/Vue) и упаковывать его в нативное нативное окно с Go backend.  
**Последствия:**  
  - Positive: один бинарник (exe); нативное окно (без браузера); доступ к Go-функциям напрямую через bindings.  
  - Negative: размер бинарника больше (включает WebView2 runtime); необходимо поддерживать фронтенд-сборку.  
**Альтернативы:**  
  - Fyne — чистый Go UI, не требует WebView. Слабее экосистема, сложнее сделать читаемый гейминг-интерфейс. Отклонено.  
  - Gio (Gioui) — чистый Go, хорош, но меньше документации.  
  - Electron — перебор для pet-проекта.  
  - Tauri — Rust-based, противоречит цели "изучить Go".  
**Связанные ADR:** ADR-005 (remove HTTP/WS)
