# ADR-002: Выбор Go как основного языка

**Статус:** Active  
**Дата:** 2026-07-17  
**Контекст:** Основная цель проекта — изучение Go. Требуется язык с хорошей поддержкой конкурентности, простым деплоем (один бинарник) и богатой стандартной библиотекой.  
**Решение:** Go 1.22 выбран как основной язык для orchestrator, advisor, storage и UI-слоя (через Wails).  
**Последствия:**  
  - Positive: один бинарник для core-компонента; отличная стандартная библиотека (slog, net/http, testing); goroutines для конкурентных pipeline.  
  - Negative: Go не является основным языком для ML — Python используется в офлайн-контуре для обучения моделей.  
**Альтернативы:**  
  - Rust — круче, но не цель проекта.  
  - C# — неплох для Windows Desktop, но не соответствует цели "изучить Go".  
  - Python — отличный для ML/CV, но не для нативного Desktop и не для изучения Go.  
**Связанные ADR:** ADR-003 (Python только для CV), ADR-006 (Wails)
